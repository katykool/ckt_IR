# ckt_IR

## Корпус
 
[HSE-Chukchi-NLP/russian-chukchi-parallel-corpora](https://huggingface.co/datasets/HSE-Chukchi-NLP/russian-chukchi-parallel-corpora) — параллельный корпус из ~70k пар предложений (чукотский + русский перевод)
 
Корпус фильтруется при загрузке:
- удаляются строки с пустым текстом
- удаляются однословные предложения
- удаляются пары с низким качеством выравнивания (score < 0.6)
 
После фильтрации остаётся ~30k документов.

## Структура проекта
 
```
├── pipeline.py       # загрузка корпуса, токенизация, индексирование, поиск
├── main.py            # точка входа 
├── requirements.txt  # зависимости
└── README.md
```

## Модели
 
### BM25
Лексический поиск на основе TF-IDF с нормализацией длины документа. Не требует обучения, работает по точному вхождению токенов. Реализован через библиотеку `rank_bm25`.
 
### Word2Vec
Нейронная модель, обучающая векторные представления слов на основе контекста. Обучается с нуля на чукотском корпусе через `gensim`. Вектор документа — нормализованное среднее векторов его токенов. Поиск — косинусное сходство.
 
Параметры: `vector_size=50`, `window=3`, `min_count=1`, `epochs=10`.
 
### Random Indexing
По сути аналог word2vec. Метод векторного представления слов основанный на случайных разреженных проекциях. Каждому документу назначается случайный разреженный вектор (сигнатура), вектор слова — сумма сигнатур всех документов, где оно встречается. Вектор документа при поиске — нормализованное среднее векторов его слов.
 
Не требует большого корпуса для обучения, поэтому особенно подходит для низкоресурсных языков. Поиск — косинусное сходство.
 
Параметры: `dim=1000`, `nonzero=6`.

### FastText

Разбивает слова на символьные n-граммы и обучает векторы для каждой из них. Вектор слова — сумма векторов его n-грамм, это позволяет строить представления для слов не встречавшихся в обучающих данных. Обучается с нуля на чукотском корпусе через `gensim`. Вектор документа — нормализованное среднее векторов его токенов. Поиск — косинусное сходство.

Параметры: `vector_size=50`, `window=3`, `min_count=1`, `epochs=10`.

Несмотря на то что такой n-граммный теоретически хорошо подходит для полисинтетических языков, на корпусе данного размера FastText показывает плохой результат. Документы слабо различаются, видимо, из-за того, что n-граммы чукотских слов слишком часто пересекаются.

### Сравнение моделей
 
На корпусе данного размера (~30k предложений) лексический поиск **BM25**, **Word2Vec** и **Random Indexing** показывают наилучшие результаты. Модели типа FastText не дают достаточной дифференциации между документами из-за малого размера обучающих данных.

## Установка
 
```bash
pip install -r requirements.txt
```
 
## Использование
 
```bash
python main.py "ӄораӈы"
```
 
#### Параметры
 
| Параметр | Описание | Значение по умолчанию |
|---|---|---|
| `query` | Поисковый запрос на чукотском | обязательный |
| `--index` | Тип индекса: `bm25`, `w2v`, `ri` | `bm25` |
| `--top_k` | Количество результатов | `5` |
 
#### Примеры
 
```bash
# поиск с BM25
python cli.py "ӄораӈы"
 
# поиск с Word2Vec
python cli.py "ӄораӈы" --index w2v
 
# поиск с Random Indexing, 10 результатов
python cli.py "ӄораӈы" --index ri --top_k 10
```
 
### Программный интерфейс
 
```python
from pipeline import load_corpus, build_indices, search, print_results
 
doc_texts_ckt, doc_texts_ru = load_corpus()
indices = build_indices(doc_texts_ckt)
 
results = search(
    query="ӄораӈы",
    index="bm25",
    indices=indices,
    doc_texts_ckt=doc_texts_ckt,
    doc_texts_ru=doc_texts_ru,
    top_k=5
)
print_results(results)
```
