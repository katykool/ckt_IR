# Chukchi Information Retrieval
 
Система поиска по чукотскому корпусу с исправлением опечаток и рекомендательной системой на основе газетных статей. Документы — предложения из параллельного русско-чукотского корпуса, результаты поиска отображаются с русским переводом.
 
## Идея проекта
 
Пользователь ищет текст на чукотском языке, а система не только возвращает релевантные предложения из корпуса, но и предлагает газетные статьи об этнографии и культуре чукчей, связанные с запросом. 

## Языковой корпус
 
[HSE-Chukchi-NLP/russian-chukchi-parallel-corpora](https://huggingface.co/datasets/HSE-Chukchi-NLP/russian-chukchi-parallel-corpora) — параллельный корпус из ~70k пар предложений (чукотский + русский перевод). Используется для индексирования и поиска.
 
Корпус фильтруется при загрузке:
- удаляются строки с пустым текстом
- удаляются однословные предложения
- удаляются пары с низким качеством выравнивания (score < 0.6)
 
После фильтрации остаётся ~30k документов.

## Газетный корпус
`nutenut_news.xlsx`: статьи газеты «Крайний Север» на чукотском и русском языках из рубрики [Мургин нутэнут](https://ks87.ru/articles/murgin-nutenut/). Используется для рекомендательной системы.

## Модели
 
### BM25
Лексический поиск на основе TF-IDF с нормализацией длины документа. Не требует обучения, работает по точному вхождению токенов. Реализован через библиотеку `rank_bm25`.
 
### Word2Vec
Нейронная модель, обучающая векторные представления слов на основе контекста. Обучается с нуля на чукотском корпусе через `gensim`. Вектор документа — нормализованное среднее векторов его токенов. Поиск — косинусное сходство.
 
Параметры: `vector_size=50`, `window=3`, `min_count=1`, `epochs=10`.
 
### Random Indexing
По сути аналог word2vec. Метод векторного представления слов основанный на случайных разреженных проекциях. Каждому документу назначается случайный разреженный вектор (сигнатура), вектор слова — сумма сигнатур всех документов, где оно встречается. Вектор документа при поиске — нормализованное среднее векторов его слов.
 
Не требует большого корпуса для обучения, поэтому особенно подходит для низкоресурсных языков. Поиск — косинусное сходство.
 
Параметры: `dim=1000`, `nonzero=6`.

### FastText

Разбивает слова на символьные n-граммы и обучает векторы для каждой из них. Вектор слова — сумма векторов его n-грамм, это позволяет строить представления для слов не встречавшихся в обучающих данных. Обучается с нуля на чукотском корпусе через `gensim`. Вектор документа — нормализованное среднее векторов его токенов. Поиск — косинусное сходство.

Параметры: `vector_size=50`, `window=3`, `min_count=1`, `epochs=10`.

Несмотря на то что такой n-граммный теоретически хорошо подходит для полисинтетических языков, на корпусе данного размера FastText показывает плохой результат. Документы слабо различаются, видимо, из-за того, что n-граммы чукотских слов слишком часто пересекаются.

### Сравнение моделей
#### Однословный запрос
BM25 находит точные вхождения без ранжирования (скоры примерно одинаковые), Word2Vec и Random Indexing возвращают релевантные результаты (RI с лучшей дифференциацией скоров), FastText выдаёт нерелевантные результаты из-за широкого пересечения n-грамм на малом корпусе.
#### Многословный запрос
BM25 хорошо ранжирует документы с обоими токенами выше чем с одним. RI на многословных запросах работает заметно лучше чем на однословных.

## Исправление опечаток
 
Реализован на основе расстояния Левенштейна и частот слов корпуса (алгоритм Норвига). Для каждого токена запроса ищется ближайшее слово из словаря корпуса (сначала edit distance 0, затем 1, затем 2). Среди кандидатов выбирается наиболее частотное слово.

## Рекомендательная система
 
При поиске по корпусу система предлагает релевантные газетные статьи о соответствующих культурных практиках и традициях чукчей. Реализована на основе TF-IDF + косинусном сходстве между запросом и текстами статей.

pip install -r requirements.txt

## Использование
 
```bash
python main.py "ӄораӈы"
```
 
#### Параметры
 
| Параметр | Описание | Значение по умолчанию |
|---|---|---|
| `query` | Поисковый запрос на чукотском | обязательный |
| `--mode` | Режим: `search`, `recommend`, `all` | `all` |
| `--index` | Тип индекса: `bm25`, `w2v`, `fasttext`, `ri` | `bm25` |
| `--top_k` | Количество результатов | `5` |
| `--spellcheck` | Исправить опечатки перед поиском | выключено |
| `--news_path` | Путь к файлу газетного корпуса | `nutenut_news.xlsx` |
 
 
#### Примеры
 
```bash
# поиск + рекомендации (по умолчанию)
python main.py "ӄораӈы"
 
# только поиск
python main.py "ӄорамы" --mode search
 
# только поиск с выбором индекса
python main.py "ӄораӈы" --mode search --index ri
 
# только рекомендации
python main.py "ӄораӈы" --mode recommend
 
# всё вместе со спеллчекером
python main.py "ӄорамы" --mode all --spellcheck --index bm25
```

### Программный интерфейс
 
```python
from pipeline import load_corpus, build_indices, search, print_results, tokenize
from spellchecker import SpellChecker, build_vocab
from recommender import load_news, build_news_index, recommend
 
# загрузка и индексирование
doc_texts_ckt, doc_texts_ru = load_corpus()
doc_tokens = [tokenize(text) for text in doc_texts_ckt]
indices = build_indices(doc_texts_ckt)
 
# спеллчекер
spell = SpellChecker(build_vocab(doc_tokens))
query = spell.correct_query("ӄораӈы")
 
# поиск
results = search(
    query=query,
    index="bm25",
    indices=indices,
    doc_texts_ckt=doc_texts_ckt,
    doc_texts_ru=doc_texts_ru,
    top_k=5
)
print_results(results)
 
# рекомендации
news = load_news("nutenut_news.xlsx")
tfidf, matrix = build_news_index(news)
recommend(query, news, tfidf, matrix, top_k=3)
```

 
Каждый результат содержит оценку близости, исходное предложение на чукотском и русский перевод.
